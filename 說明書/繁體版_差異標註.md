# 原始版與新版差異標註

比較對象：

- 原始：`my_project/pudu-robot-control/Pudu_Cloud_API_Docs拷貝.md`
- 新版：`PYTHON API/Pudu_Cloud_API_繁體使用說明書_整合版.md`

---

## 1. 文件定位差異

- 原始版是官方 API 文件彙整（章節多、內容長、偏參考手冊）。
- 新版是可執行導向的繁體整合版（優先串接流程、參數來源路徑、錯誤避坑）。

## 2. 語言與用詞差異

- 新版統一繁體中文用字（例如「設定、顯示、機器、說明」）。
- 新版補齊臺灣常用描述，降低簡繁混用造成的閱讀成本。

## 3. 圖片錯誤修正差異

- 修正 `group_by` 說明：
  - 原始多處誤寫為時區描述。
  - 新版統一為「分組方式（`robot` / `shop`）」。
- 修正清潔統計 `sub_mode` 說明：
  - 原始說明不一致且容易誤解。
  - 新版統一為常用過濾值：`-1/0/1/3`（並保留 `clean_mode` 搭配說明）。

## 4. 你指定 API 的覆寫差異

- `# 取得點位分組`
  - 原始版偏向限定特定機型語意。
  - 新版依你的需求標註為「每臺都支持」。
- `# 設置機器屏幕`
  - 原始版描述為舊介面且支持範圍保守。
  - 新版依你的需求明確納入「FlashBot Max 可使用」。

## 5. 特殊規則整合差異

- 新版把 `API_特殊需求說明.md` 重要規則合併進主手冊：
  - 網站內群組兼容 vs 官方直打差異
  - `v1/v2` 狀態與回充使用建議
  - `map_name`、時間戳處理規則

## 6. 參數來源路徑差異（新版新增）

- 新版新增每個關鍵參數的「先打哪支 API 拿值」：
  - `shop_id`、`sn`、`group_id`、`map_name`
- 並提供一條可直接照做的串接順序（先查清單，再打目標 API）。

## 7. 可用性差異

- 原始版適合查完整官方欄位與範例。
- 新版適合工程與現場快速落地，減少「不知道參數要去哪拿」的時間。
